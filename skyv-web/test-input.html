测试执行间隔输入框问题

问题描述：选择周期执行时，执行间隔框无法输入数据

已修复的问题：

  • 将 el-input-number 替换为 el-input type="number"
  • 添加了输入验证函数 handleIntervalValueInput
  • 修复了调度类型切换时的数据重置逻辑
  • 添加了watch监听器用于调试
  • 添加了输入范围限制（1-9999）

测试步骤：

  1. 打开任务创建页面
  2. 完成基本信息和设备选择
  3. 在调度设置页面选择"周期执行"
  4. 尝试在"执行间隔"输入框中输入数字
  5. 检查浏览器控制台是否有 "intervalValue changed" 日志输出

预期结果：

  • 输入框可以正常输入数字
  • 输入值在1-9999范围内
  • 实时显示"任务将每隔 X 分钟执行一次"
